# Webhooks

Push beats pull. Subscribe to events and Nexa POSTs to your URL the moment
something happens â€” job finished, campaign created, credits topped up, creator
profile enriched.

## Subscribe

Under **Settings â†’ Webhooks** in any product dashboard:

1. Add your endpoint URL (HTTPS only).
2. Pick events. `*` subscribes to everything.
3. Copy the signing secret â€” you will need it to verify deliveries.

## Event envelope

## Verify signatures

Every delivery carries `X-Nexa-Signature` â€” HMAC-SHA256 of the raw request
body using your webhook secret, with timestamp protection.

Header format:

```
X-Nexa-Signature: t=1713441600,v1=ae2c...
```

Reject if the timestamp is older than 5 minutes, or if the HMAC doesn't match.

```ts
import crypto from "node:crypto";

export function verify(req: { rawBody: Buffer; headers: Record<string,string> }, secret: string) {
  const header = req.headers["x-nexa-signature"] ?? "";
  const parts = Object.fromEntries(header.split(",").map(kv => kv.split("=")));
  const t = Number(parts.t);
  if (Math.abs(Date.now()/1000 - t) > 300) return false;

  const payload = `${t}.${req.rawBody.toString("utf8")}`;
  const expected = crypto.createHmac("sha256", secret).update(payload).digest("hex");
  const received = Buffer.from(parts.v1, "hex");
  const want = Buffer.from(expected, "hex");
  return received.length === want.length && crypto.timingSafeEqual(received, want);
}
```

```python
import hmac, hashlib, time

def verify(raw_body: bytes, header: str, secret: str) -> bool:
    parts = dict(kv.split("=") for kv in header.split(","))
    t = int(parts["t"])
    if abs(time.time() - t) > 300:
        return False
    payload = f"{t}.".encode() + raw_body
    expected = hmac.new(secret.encode(), payload, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, parts["v1"])
```

```go
import (
  "crypto/hmac"
  "crypto/sha256"
  "encoding/hex"
  "strconv"
  "strings"
  "time"
)

func Verify(body []byte, header, secret string) bool {
  parts := map[string]string{}
  for _, kv := range strings.Split(header, ",") {
    if k, v, ok := strings.Cut(kv, "="); ok { parts[k] = v }
  }
  t, _ := strconv.ParseInt(parts["t"], 10, 64)
  if abs(time.Now().Unix()-t) > 300 { return false }
  mac := hmac.New(sha256.New, []byte(secret))
  mac.Write([]byte(strconv.FormatInt(t,10)+".")); mac.Write(body)
  want := hex.EncodeToString(mac.Sum(nil))
  return hmac.Equal([]byte(want), []byte(parts["v1"]))
}
```

## Delivery semantics

- **Timeout.** You have 10 seconds to respond `2xx`.
- **Retries.** On non-`2xx` or timeout we retry with exponential backoff:
  30s, 2m, 10m, 1h, 6h, 24h, 3d (max 7 attempts).
- **Ordering.** Not guaranteed â€” two deliveries for the same resource may
  arrive out of order. Use `data.object.updated_at` to resolve.
- **Idempotency.** Deduplicate on `id` (the `evt_...`). We may deliver the
  same event twice after a retry storm.

<Callout>
  Respond `200 OK` fast, enqueue the work. If your handler takes >10 s you will
  get hammered with retries â€” and the user calls it a bug.
</Callout>

## Test mode

In sandbox (test keys), webhooks fire for **simulated** jobs that complete in
~500 ms. Use them in CI.

```bash
curl https://api.nexacreate.com/v1/webhooks/test \
  -H "Authorization: Bearer $NEXA_TEST_KEY" \
  -d '{"event":"image.job.succeeded"}'
```

## Security tips

1. Put the secret in a vault, not in code.
2. Always verify the signature **before** any downstream work.
3. Rate-limit your own endpoint â€” a compromised secret shouldn't DoS you.
4. Rotate the secret yearly (one-click in the dashboard).
