# NexaAds API

<span className="nexa-product-pill" data-product="ads">NexaAds</span>

NexaAds exposes two complementary surfaces:

1. **REST API** — for humans and backend integrations.
2. **MCP server** — for LLM agents (Claude, Cursor, OpenAI Agents, …).

Both share the same auth, the same resources, the same rate limits.

## When to use what

| I want to…                           | Use                                |
| ------------------------------------ | ---------------------------------- |
| Build a CRUD dashboard               | **REST**                           |
| Let Claude manage ads in chat        | **MCP**                            |
| Run a scheduled budget optimizer     | **REST** (cron) + `budget.optimize`|
| Expose one endpoint to a customer    | **REST** with scoped token         |

<Callout>
  MCP tool names mirror REST verbs — `nexaads.campaigns.create` is the MCP
  equivalent of `POST /v1/campaigns`. Learn one, you know the other.
</Callout>
