## 5-minute quickstart

<Tabs items={["TypeScript", "Python", "cURL"]}>
  <Tabs.Tab>

```ts
import { Nexa } from "@nexa/sdk";

const nexa = new Nexa({ apiKey: process.env.NEXA_API_KEY! });

const image = await nexa.create.images.generate({
  prompt: "A minimalist product shot of a pink sneaker on violet gradient",
  aspect_ratio: "1:1",
});

console.log(image.url);
```

  </Tabs.Tab>
  <Tabs.Tab>

```python
from nexa import Nexa

nexa = Nexa(api_key=os.environ["NEXA_API_KEY"])

image = nexa.create.images.generate(
    prompt="A minimalist product shot of a pink sneaker on violet gradient",
    aspect_ratio="1:1",
)

print(image.url)
```

  </Tabs.Tab>
  <Tabs.Tab>

```bash
curl https://api.nexacreate.com/v1/images \
  -H "Authorization: Bearer $NEXA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "A minimalist product shot of a pink sneaker on violet gradient",
    "aspect_ratio": "1:1"
  }'
```

## Why Nexa

- **One token, every product.** [`nexa-auth`](/auth) issues a single bearer that scopes across Ads, Create, Lytics, Voize.
- **Agent-native.** Every product ships an MCP server — Claude, Cursor, and OpenAI Agents connect with a single URL.
- **Predictable shapes.** Same error format, same pagination, same idempotency rules everywhere.
- **Typed SDKs.** First-class TypeScript, Python, and Go with retries, timeouts, and structured errors built in.
